VLAN Configuration

A VLAN is a technology used to divide a single physical switch into multiple independent virtual switches. This division is configured logically (via software) rather than physically.

The Problem it Solves

By default, all ports on a switch belong to a single network, known as a Broadcast Domain. This means that if one device sends a broadcast message, it reaches every other device connected to the switch. This leads to network congestion and weakens security, as all departments can access the same network space.

How Does a VLAN Work?

This technology isolates different departments from each other. For example, we can assign ports 1 to 10 to the Human Resources department (VLAN 10), and ports 11 to 20 to the IT department (VLAN 20). By doing this, devices in VLAN 10 cannot communicate with devices in VLAN 20—even though they are connected to the exact same physical switch—unless a Router or a Layer 3 Switch is used for Inter-VLAN Routing.

Key Benefits in Enterprise Networks

  • Enhanced Security: Isolates sensitive departments from the rest of the network, preventing unauthorized users from accessing restricted data.
  • Improved Performance: Reduces the size of the broadcast domain, which minimizes random network traffic and saves bandwidth.
  • Flexibility & Cost Reduction: Instead of purchasing a separate physical switch for every department, you can use a single switch and logically divide it to meet everyone's needs.

Step-by-Step CLI Configuration

A complete VLAN configuration in the real world requires three main steps: creating the VLAN, assigning an interface to it, and finally verifying the setup.

Switch# configure terminal Switch(config)# vlan 10 Switch(config-vlan)# name Sales Switch(config-vlan)# exit Switch(config)# interface FastEthernet 0/1 Switch(config-if)# switchport mode access Switch(config-if)# switchport access vlan 10 Switch(config-if)# end Switch# show vlan brief
